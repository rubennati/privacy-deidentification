# Rules

- Keep work scoped to the stated objective.
- Preserve human reviewability; keep changes small and focused.
- Avoid unrelated refactors.
- Record meaningful state changes in `state.md`.
- Record architecture decisions in `decisions.md` (ADR under `docs/adr/`).
- Escalate major design or dependency changes for human approval.
- Never commit secrets. Config comes from environment variables.
