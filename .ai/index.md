# AI Workspace Index

Operational context for AI-assisted work on this project. Keep concise and low-overhead.

## Continuing existing work? ("weiter" / "machen wir weiter")

Read **`state.md`** first — its top is the current snapshot (where we are, current focus/next, and the
important reversals), and its end holds the condensed milestone log. Then **`tasks.md`** (open work) and
**`decisions.md`** (decisions **and rejected paths** — why a discarded approach should not be re-tried).
That is enough to resume without re-deriving context.

## Start sequence (first-time / setup)

1. `project-brief.md`
2. `state.md`
3. `routing.md`
4. `rules.md`

## Core files

- Execution status: `state.md` (current snapshot + condensed history), `tasks.md`
- Governance memory: `decisions.md` (incl. rejected paths), `quality-gates.md`
- Working patterns: `domains/`

See `AGENTS.md` at the repo root for the authoritative workflow and approval rules.
